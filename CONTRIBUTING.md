# Contributing to NotepadSharp

Thanks for helping make NotepadSharp great.

## Quick start
- Fork the repo and create a branch from `main`
- Build: `dotnet restore` then `dotnet build`
- Test: `dotnet test`

## What to work on
- Check GitHub Issues (especially items labeled `good first issue`)
- If youâ€™re proposing a big change, open an issue first to align on scope

## Coding guidelines
- Keep PRs focused and small when possible
- Add/adjust tests for behavior changes
- Keep cross-platform behavior in mind (Windows/macOS/Linux)

## Commit / PR notes
- Describe the "why" and include screenshots for UI changes
- Ensure CI is green

## Reporting security issues
Please do not file public issues for security vulnerabilities. See `SECURITY.md`.
